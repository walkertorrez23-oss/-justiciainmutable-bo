Fundamentación Jurídica

La Base Legal ya existe

Un buen proyecto se ancla en la normativa vigente. Para implementar Blockchain judicial, contamos con tres pilares legales bolivianos.

📜
CPE Art. 162

Iniciativa Legislativa

La facultad de proponer leyes la tienen ciudadanos, asambleístas, el Ejecutivo y el Tribunal Supremo (en materia judicial). La vía ideal es el Tribunal Supremo de Justicia o Asambleístas de la Comisión de Justicia.

💻
Ley N° 164

Telecomunicaciones y TIC

Otorga validez jurídica a documentos y firmas digitales. Es la piedra angular. El Blockchain actuará como notario digital distribuido que certifica las firmas generadas bajo esta ley.

⚖️
Ley N° 1173

Abreviación Procesal

Ordena el Sistema Informático de Gestión de Causas y el Expediente Electrónico. El Blockchain es la capa de seguridad subyacente para evitar alteraciones por consorcios.

¿Por qué Blockchain?

La prueba matemática de la transparencia

El problema actual: pérdida de expedientes, alteración de folios, mora procesal y consorcios. Con Blockchain, una vez registrado un actuado, nadie puede modificarlo sin dejar rastro evidente.

  • ✔Inmutabilidad: Adiós a la "pérdida de fojas".
  • ✔Trazabilidad: Se sabe exactamente quién firmó qué y a qué hora.
  • ✔Auditoría pública: Cualquier ciudadano puede verificar la integridad del expediente.

Arquitectura & Código

Del Papel (Ley) al Código

Un artículo que dice "el expediente será inmodificable y registrará al autor" se traduce en un Smart Contract dentro de una red Blockchain permisionada (Hyperledger Fabric o Ethereum privado).

Solidity — Contrato Inteligente Simula el "Art. 3: Registro Inmutable" 
// Contrato Inteligente que representa un Expediente Judicial Boliviano
pragma solidity ^0.8.0;

contract JusticiaBolivia {

  // Estructura de un Documento (Memorial, Providencia, Sentencia)
  struct ActuadoJudicial {
    string  hashDocumento; // Hash IPFS del PDF — garantiza inalterabilidad
    uint256 fechaHora;    // Timestamp inmutable en blockchain
    address emisor;       // Billetera (Firma digital) del Juez/Abogado
    string  tipoActuado;  // Ej: "Sentencia", "Memorial", "Providencia"
  }

  // Mapeo: NUREJ boliviano → lista de actuados del expediente
  mapping(string => ActuadoJudicial[]) public expedientes;

  // Evento para auditoría pública al registrar un nuevo documento
  event NuevoActuadoRegistrado(string nurej, string hashDoc, address emisor);

  // Equivale al acto de "Ingresar memorial por buzón judicial"
  function registrarActuado(
    string memory _nurej,
    string memory _hashDoc,
    string memory _tipo
  ) public {
    expedientes[_nurej].push(ActuadoJudicial({
      hashDocumento: _hashDoc,
      fechaHora:     block.timestamp,
      emisor:        msg.sender,
      tipoActuado:   _tipo
    }));
    emit NuevoActuadoRegistrado(_nurej, _hashDoc, msg.sender);
  }
}
Node.js — Backend Integración con Ley 164 (Firma Digital ADSIB) 
const crypto         = require('crypto');
const { ethers }      = require('ethers');          // Interacción blockchain
const validadorADSIB = require('./servicios/adsib'); // Validación Ley 164

async function subirMemorialAlExpediente(nurej, archivoPDF, firmaDigitalPem) {
  try {
    // 1. Validar firma digital bajo la Ley 164 boliviana
    const esFirmaValida = await validadorADSIB.verificar(archivoPDF, firmaDigitalPem);
    if (!esFirmaValida) throw new Error("Firma digital inválida según ADSIB");

    // 2. Hash SHA-256 del PDF — si se cambia una coma, el hash cambia
    const hashBuffer    = crypto.createHash('sha256').update(archivoPDF).digest('hex');
    const hashDocumento = `0x${hashBuffer}`;

    // 3. Conectar al Smart Contract desplegado
    const contrato = new ethers.Contract(direccionContrato, abi, walletAutorizada);

    // 4. Ejecutar transacción en la Blockchain
    const tx = await contrato.registrarActuado(nurej, hashDocumento, "Memorial");
    await tx.wait(); // Esperar confirmación de la red

    return { exito: true, txHash: tx.hash, documentoHash: hashDocumento };

  } catch (error) {
    console.error("Error procesando actuado:", error);
    return { exito: false, error: error.message };
  }
}

// Lógica de UI para auditar un expediente desde la interfaz del juez
async function auditarExpediente(nurej) {
  const historial    = await getHistorialDesdeContrato(nurej);
  const contenedor   = document.getElementById('lista-actuados');
  contenedor.innerHTML = '';

  historial.forEach(actuado => {
    const el      = document.createElement('div');
    el.className = 'actuado-card border p-4 mb-2 rounded';

    // El Hash es la prueba matemática de que el documento es auténtico
    el.innerHTML = `
      <h4 class="font-bold">${actuado.tipoActuado}</h4>
      <p><b>Fecha:</b> ${new Date(actuado.fechaHora * 1000).toLocaleString()}</p>
      <p><b>Firmante:</b> ${actuado.emisor}</p>
      <div class="hash-box">
        <span class="badge-ok">Hash de Integridad ✔</span>
        <code>${actuado.hashDocumento}</code>
      </div>
    `;
    contenedor.appendChild(el);
  });
}
